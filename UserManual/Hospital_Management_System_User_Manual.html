1. Introduction

The Hospital Management System (HMS) is a comprehensive web-based application designed to manage and streamline hospital operations. This system facilitates the management of various hospital activities including patient registration, appointment scheduling, doctor management, and administrative tasks.

2. System Requirements

Minimum Requirements:

  • Web Server: XAMPP/WAMP/MAMP
  • PHP Version: 7.2.4 or higher
  • MySQL Version: 5.7 or higher
  • Web Browser: Modern browsers (Chrome, Firefox, Safari)
  • Internet Connection: Required for external resources

3. User Roles

Role Access Level Responsibilities
Patient Basic
  • Register account
  • Book appointments
  • View medical history
  • View prescriptions
Doctor Intermediate
  • View appointments
  • Manage patient records
  • Write prescriptions
  • Update patient status
Admin Advanced
  • Manage doctors
  • Manage patients
  • View system reports
  • System configuration

4. Getting Started

4.1 Accessing the System

  1. Open your web browser
  2. Enter the system URL (localhost/Hospital-Management-System)
  3. Select your user role (Patient/Doctor/Admin)
  4. Login with your credentials

4.2 First-Time Login

For new patients:

  1. Click on "Register" on the home page
  2. Fill in required personal information
  3. Create a password (minimum 6 characters)
  4. Submit registration form
  5. Login with registered email and password

5. Patient Module

5.1 Booking an Appointment

  1. Login to patient account
  2. Click "Book Appointment"
  3. Select preferred doctor
  4. Choose available date and time
  5. Confirm booking

5.2 Viewing Medical History

  • Access appointment history
  • View prescriptions
  • Download medical reports

9. Troubleshooting

Issue Solution
Unable to login
  • Check username and password
  • Clear browser cache
  • Contact admin if issue persists
Appointment booking failed
  • Check internet connection
  • Verify doctor availability
  • Try booking for a different time slot
System not loading
  • Check server status
  • Verify database connection
  • Contact system administrator
